# 清墨 0.3.3 发布清单

## 功能与数据安全

- [x] 新建、打开、保存、另存为及真实 Markdown 文件流程
- [x] 撤销与重做（工具栏和快捷键）
- [x] 同目录临时文件、刷盘与原子替换保存
- [x] 外部文件修改冲突检测和用户确认
- [x] 未命名及已有文档的崩溃恢复草稿
- [x] 关闭前保存期间继续输入时，不会误退出并丢失最后输入
- [x] 放弃更改后，只有新文档成功替换才清理恢复草稿
- [x] 双击 Markdown 启动时，先打开指定文件，不与恢复草稿异步竞争
- [x] 菜单和快捷键重新加载会走未保存确认
- [x] 启动文件与旧恢复草稿明确二选一，不再静默清除草稿
- [x] 保存和恢复文件操作串行化，避免旧操作晚完成
- [x] 用户确认覆盖后再次校验最新磁盘指纹
- [x] UTF-8 BOM、UTF-16 LE/BE、GB18030 检测与保存
- [x] 原编码、BOM 和 LF/CRLF/CR 换行保留
- [x] 远程资源旁路全部清理，点击后仅加载公开 HTTPS 位图
- [x] 不确定编码保存前警告，并阻止明显二进制文件
- [x] 亮色和暗色语法高亮配色
- [x] 禁止新窗口和页面导航，保留沙箱及 HTML 清洗

## 构建与安装

- [x] 版本统一取自 `package.json`（0.3.3）
- [x] Electron 本地 ZIP 缓存可选，干净克隆可联网构建
- [x] 图标完全使用 Node.js 生成，不依赖 Python/Pillow
- [x] 生产包排除测试、源码、构建脚本和安装脚本
- [x] 当前用户中文安装器、可选桌面快捷方式与“打开方式”
- [x] 本机提供中文文件名；GitHub Release 提供 `QingMo` 与 `CleanMark` 两个同内容安装器
- [x] 构建脚本同步生成英文发布资产和当前版本 `SHA256SUMS.txt`
- [x] 安装版冒烟与完整 E2E 通过，静默卸载后目录、开始菜单和卸载项均清理
- [x] 正常卸载可选择清除设置和明文恢复草稿
- [x] 安装器 SHA-256 校验值随本地构建和 GitHub Release 生成
- [ ] Windows 代码签名（需要开发者提供有效代码签名证书）
