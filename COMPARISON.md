# 清墨 0.4.0 功能对照与不足

本文件依据成熟 Markdown 编辑器的公开功能规格以及清墨的实际代码、测试和打包状态进行对照。它只讨论功能，不表示兼容实现，也不复用 Typora 的代码、资源、字节码或专有桥接。

## 与成熟编辑器相比仍有不足

| 能力 | 优先级 | 当前边界 |
| --- | --- | --- |
| 多标签、多窗口、最近文件 | 高 | 一次只编辑一个文档 |
| 多版本恢复历史 | 高 | 目前保留最近一次崩溃恢复草稿 |
| 外部变更实时监听和内容合并 | 高 | 保存前会保护性检查，但没有三方合并界面 |
| 批量或文件夹转换 | 高 | 0.4.0 仅支持单文件 |
| PDF 可编辑结构恢复 | 高 | PDF 缺少语义结构，只能尽力恢复 |
| 复杂 Office 对象恢复 | 中 | 动画、宏、SmartArt、嵌入对象和特殊字体可能降级 |
| 数学公式、Mermaid 和流程图 | 中 | 暂无完整编辑、预览和跨格式保真链路 |
| 图片插入、复制、移动和资源管理 | 中 | 主要解析已有图片引用 |
| 大纲、专注和打字机模式 | 中 | 暂无长文写作辅助视图 |
| 单窗口所见即所得编辑 | 中 | 当前为源码编辑加预览 |
| 自定义主题 CSS 和快捷键 | 低 | 当前提供固定主题和快捷键 |
| 自动更新和代码签名 | 发布能力 | 尚未实现自动更新，安装器未商业签名 |
| macOS/Linux 发行 | 按需 | 当前只验证 Windows x64 |

## 转换功能的实际定位

清墨 0.4.0 已不再缺少基础 PDF、DOCX 和 PPTX 导入导出，但“能转换”不等于所有格式都能无损互转：

- Markdown、DOCX、PPTX 的结构化内容更适合“可编辑”模式。
- PDF 转可编辑格式取决于原 PDF 的文字层、阅读顺序和表格结构。
- 复杂页面和幻灯片优先使用“视觉保真”模式。
- Office 可用时，Word/PPT 直接导出 PDF 通常比内置重建更接近原排版。
- 清墨不会宣称恢复动画、宏逻辑或任意专有嵌入对象。

完整说明见 [CONVERSION_QUALITY.md](CONVERSION_QUALITY.md)。

## 结论

清墨已能可靠完成基础 Markdown 写作和本地单文件格式转换，但仍不是成熟商业编辑器的等价替代。下一阶段最有价值的方向是多标签、最近文件、版本历史、外部变更合并，以及继续提高复杂表格、公式和 Office 布局的恢复质量。
