# 清墨与原项目的功能差距

对照依据是原项目安装包随附的功能文档，以及清墨 0.3.2 的实际测试与打包状态。
该比较仅讨论功能，不复用原项目代码。

## 已实现并通过发布版测试

| 功能 | 清墨状态 | 验证方式 |
| --- | --- | --- |
| 新建、打开、覆盖保存、另存为、重开 | 已实现 | 开发版 E2E + 打包 EXE + Windows 原生对话框 |
| 中文 UTF-8 Markdown | 已实现 | 中文文件名、正文和 JSON 报告均落盘验证 |
| GFM 常用语法 | 已实现 | 标题、粗体、斜体、删除线、列表、任务、表格、引用、代码块 |
| 实时预览和代码高亮 | 已实现 | DOM 断言及发布版可访问性树验证 |
| 文档内标题锚点 | 已实现 | 中文锚点生成与目标匹配测试 |
| 相对路径图片 | 已实现 | 相对 Markdown 文件目录解析 SVG |
| 文件夹树、嵌套目录、筛选 | 已实现 | 真实多层测试目录验证 |
| 编辑、分屏、预览 | 已实现 | 三种布局断言 |
| 自动保存 | 已实现 | 等待防抖后重新从磁盘读取 |
| 保存期间继续输入 | 已修复 | 验证不会错误清除未保存状态 |
| 切换文档时隔离自动保存 | 已修复 | 文档令牌取消旧定时任务 |
| 未保存关闭保护 | 已实现 | Windows 原生确认框，实测“取消”保留窗口 |
| 崩溃恢复草稿 | 已实现 | 未命名和已有文档恢复草稿均可读取，文档替换失败不会提前清理 |
| 外部修改冲突检测 | 已实现 | 保存前指纹检测，用户确认前不覆盖外部内容 |
| 当前用户安装器 | 已实现 | Inno Setup 生成中文安装器，不需要管理员权限 |
| HTML/XSS 清洗 | 已实现 | 注入脚本未进入预览、未执行 |
| 主题、查找、格式按钮、字词与光标统计 | 已实现 | E2E 状态和选区断言 |
| 撤销与重做 | 已实现 | 工具栏按钮、菜单及 `Ctrl+Z` / `Ctrl+Y`；回到保存基线时自动清除未保存状态 |

## 与原项目相比仍缺少

| 能力 | 优先级 | 说明 |
| --- | --- | --- |
| 未保存草稿历史、版本恢复 | 高 | 当前只有最近一次崩溃恢复草稿，没有多版本恢复 |
| 外部变更监听、二次冲突校验与合并 | 高 | 保存前已有指纹冲突提示，但用户确认覆盖到原子替换之间仍可继续加固 |
| 多标签、多窗口、最近文件 | 高 | 当前一次只编辑一个文档 |
| PDF/HTML/DOCX/EPUB 等导出 | 高 | 原项目通过内置能力和 Pandoc 支持多格式 |
| 数学公式、Mermaid/流程图 | 中 | 当前仅作为普通代码块显示 |
| 图片插入、复制、移动、重命名与上传 | 中 | 当前只解析已有相对路径图片 |
| 大纲侧栏、焦点模式、打字机模式 | 中 | 当前没有长文写作辅助视图 |
| 所见即所得单窗编辑 | 中 | 清墨是源码编辑 + 预览，不是原项目的实时 WYSIWYG 模型 |
| 拼写检查设置、语言词典 | 中 | 只使用 Chromium 基础拼写检查 |
| 自定义主题 CSS、快捷键配置 | 低 | 当前只有固定三主题和固定快捷键 |
| YAML 元数据、脚注、引用定义等高级编辑 UI | 低 | 解析器可支持其中一部分，但没有专用操作界面 |
| 打印、分页、页眉页脚 | 低 | 尚未实现出版级输出 |
| 自动更新、代码签名 | 发布前必须 | 当前 Windows 安装器未签名，自动更新尚未实现 |
| macOS/Linux 发行与测试 | 发布前按需 | 当前只验证 Windows x64 |

## 结论

清墨现在可以可靠完成基础 Markdown 文件写作流程，但不能声称达到原项目的功能完整度。
如果作为日常轻量编辑器，下一阶段应优先实现远程资源加固、外部冲突二次校验、多标签和
PDF/HTML 导出；如果目标是接近原项目，再继续增加数学、图表、图片管理和 WYSIWYG
编辑模型。
